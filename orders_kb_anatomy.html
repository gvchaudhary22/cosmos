Raw YAML is bad for embedding. Typed renderers convert it to searchable text.

Clean, semantic, high embedding quality

Step 3: Embed into Vector Database

Each rendered doc becomes a 1536-dimensional vector in pgvector

Embedding Pipeline (canonical_ingestor.py)

For each doc from Step 2:
  1. Quality Gate: reject if < 50 chars, >80% punctuation, or stub content
  2. content_hash = SHA256(content)
  3. If hash unchanged from last run → SKIP (no re-embed)
  4. embedding = OpenAI text-embedding-3-small (1536 dims)
  5. UPSERT into cosmos_embeddings table:
     - id, repo_id, entity_type, entity_id
     - content (rendered text)
     - embedding (vector)
     - metadata (JSONB: pillar, domain, chunk_type, query_mode, training_ready)
     - trust_score (0.5-0.95)
     - capability ("retrieval" | "action" | "workflow" | "routing")
     - freshness (timestamp)

For orders, this creates embeddings for:

12 schema chunks ~2,000 API chunks 11 module docs 44 action docs 13 workflow docs 1 entity hub

Each has metadata.query_mode tag: lookup act diagnose explain

async def _sync_from_graph():
    # 1. Query all tool nodes
    SELECT * FROM graph_nodes WHERE node_type = 'tool'
    → Creates: DynamicTool("orders_create", domain="orders", api_count=349)
    → Creates: DynamicTool("orders_list", domain="orders", api_count=236)
    → Creates: DynamicTool("orders_get", ...)
    → Creates: DynamicTool("orders_update", ...)

    # 2. Query all agent nodes
    SELECT * FROM graph_nodes WHERE node_type = 'agent'
    → Creates: DynamicAgent("order_ops_agent", tools={5 tools}, api_count=660)

    # 3. Query all intent nodes (= skills)
    SELECT * FROM graph_nodes WHERE node_type = 'intent'
    → Creates: DynamicSkill("orders_create", tools={"orders_create"})

    WHERE edge_type IN ('implements_tool', 'assigned_to_agent')

Anti-Hallucination Pipeline

7 layers preventing the LLM from making things up

Layer 1: Embedding Quality Gate

Reject docs < 50 chars, stubs, junk before they pollute vector search

Layer 2: Similarity Threshold 0.25

Only return chunks with ≥ 25% similarity (was 0.0 = anything)

Layer 3: Factuality Prompt

"Use ONLY facts from context. Never invent IDs. Cite sources."

Layer 4: Source Attribution

Every chunk tagged: [pillar_6:action.orders.create_order trust=0.9]

Layer 5: HallucinationGuard

BLOCK if 3+ fabricated IDs. Grounding score check.

Layer 6: RALPH Verification

Checks: response grounded in context? All intents covered?

Layer 7: "I Don't Know"

Confidence < 0.3 → "I don't have reliable information"

Layer 8: Feedback Loop

Low-confidence traces auto-generate improvement candidates
