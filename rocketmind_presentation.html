GREL — Graph-based Reasoning & Evolutionary Learning

The system that gets smarter with every query

What is GREL?

GREL is the learning layer that sits on top of the retrieval system. It continuously evaluates which strategy works best for each type of query and evolves over time.

5 Competing Strategies

StrategyMethodCost
A Decision TreeRule-based pattern matchingFree
B TF-IDF RAGKeyword retrievalLow
C Tool-UseLLM native tool selectionMedium
D Full ReasoningMulti-step ReAct loopHigh
E Hybrid RetrievalGraph + Vector + LexicalMedium

Tournament Evolution

Week 1-2: TOURNAMENT
// Run ALL strategies in parallel
// Log which one wins for each query pattern

Week 3-4: SHADOW
// Run winner + 1-2 shadows for validation
// Verify winner consistency

Month 2+: CONVERGED
// Run winner only (shadow 10% for drift)
// Cost drops 60-80% while quality holds

Learning Signals

Positive

Correct tool selected. User accepted answer. Low latency.

Negative

Wrong tool. User escalated. Timeout. Low confidence.

GraphRAG: Knowledge Graph Reasoning

Not just "find similar text" — understand relationships

Graph Structure

Nodes: api_endpoint, table, tool, agent, intent, domain, module
Edges: belongs_to_domain, implements_tool, assigned_to_agent,
       has_intent, reads_table, writes_table
Lookup: entity_type + value → node_id (O(1))

// Example traversal:
api:orders.create
  → belongs_to domain:orders
  → implements tool:orders_create
  → assigned_to agent:order_ops_agent
  → writes table:orders
  → writes table:shipments
  → has_intent intent:orders_create

Why Graph Beats Embedding-Only

Embedding-only

"cancel order" → finds docs mentioning "cancel" → might return billing docs about "cancelled invoices"

GraphRAG

"cancel order" → intent:act + entity:order → traverses to cancel_order tool → finds exact endpoint, required params, risk level, approval flow

Entity Resolution

// Query: "AWB 9876543210"
entity_lookup:
  type="awb", value="9876543210"
  → node: api:shipments.track
  → BFS 2-hop: shipments → orders → NDR
  → Full context in one traversal
